## How to Read This

**Phases:** idea → discussion → spec+plan → implementation → completed

**Step format:** `3/8` means step 3 of 8 complete

**Priority:** P0 (critical) → P1 (high) → P2 (normal) → P3 (nice-to-have)

## Feature Numbering

## Update Commands

```bash
/new-feature [name]          # start a new feature
/view-features               # refresh this view
/update-status [name]        # update after each session
/resume-feature [name]       # resume from CONTEXT.md
/complete-feature [name]     # archive and mark done
```
