# .npmignore Template

Copy this content into a `.npmignore` file at the root of any package you publish.

**Why this template?** The "obvious" pattern of excluding `*.ts` (to keep source out of the tarball) ALSO strips `.d.ts` files. That ships your package without TypeScript types and consumers silently get `any` everywhere. See `.claude/rules/publishing.md` for the full story.

---

## Recommended `.npmignore` (per package)

```
node_modules
.env
src/
*.tsbuildinfo
prisma/migrations
.git
.github
*.test.js
*.test.d.ts
*.spec.js
*.spec.d.ts
```

What this includes (i.e., what ships):
- `dist/` (your built `.js` + `.d.ts` + `.d.ts.map` + `.js.map`)
- `package.json` (npm always includes this)
- `README.md`, `CHANGELOG.md`
- `LICENSE` if present

What this excludes:
- Source TypeScript (consumers don't need it)
- env files
- TS incremental build cache
- Prisma migration history (consumers don't run your migrations)
- Git internals
- Test artifacts that may have been emitted to `dist/`

---

## Modern alternative — `package.json` `files` array

Instead of `.npmignore` (deny-list), use `files` (allow-list) in `package.json`:

```json
{
  "files": ["dist", "README.md", "CHANGELOG.md"]
}
```

Only listed paths ship. `package.json` and `LICENSE` are auto-included regardless.

Trade-off:
- **Allow-list (`files`)**: safer — anything new you add is excluded by default until you explicitly include it
- **Deny-list (`.npmignore`)**: more permissive — anything new you add ships unless you explicitly exclude it

For new packages, prefer `files`. For existing packages on `.npmignore`, the template above is safe.

---

## Verify before publishing

ALWAYS run this before `npm publish` for the first time after any tarball-shape change:

```bash
npm pack --dry-run  # show what would be published, no actual publish
# OR
npm pack             # creates @scope-pkg-X.Y.Z.tgz
tar -tzf @scope-pkg-X.Y.Z.tgz | grep '\.d\.ts$'
# Should list at least dist/index.d.ts
```

If `grep` returns nothing, stop and fix `.npmignore` / `tsconfig.json` before publishing.
